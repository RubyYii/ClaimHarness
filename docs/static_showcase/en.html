English interface

Problem alignment before AI work. Evidence auditing after outputs exist.

A local-first prototype for interdisciplinary AI projects. ProblemBridge helps teams formulate the right AI task before modeling. ClaimHarness checks whether later claims are supported by available evidence.

For non-AI users

Question Discovery Layer

When the problem is still vague, generate a question brief, stakeholder map, interview guide, unknowns list, and discussion plan before proposing a solution.

Start from work, not prompts

Describe repeated work, judgement materials, friction points, and decisions that require human review.

Get an AI-ready package

The guided app turns that description into workflow maps, task specs, evidence contracts, and evaluation protocols.

Workflow

1
Question discovery

Discover what to ask, who to ask, and what unknowns to validate.

2
Domain workflow

Capture real work, materials, judgement, and pain points.

3
ProblemBridge

Produce an aligned AI task spec, evidence contract, and evaluation protocol.

4
AI / team work

Build, write, or design with clearer boundaries.

5
ClaimHarness

Audit claims, evidence links, overclaims, and human-review needs.

Features

Question Discovery Layer

Writes a question brief, stakeholder map, expert interview guide, unknowns list, and discussion plan before solution design.

ProblemBridge

Workflow map, pain point matrix, concept alignment table, task specification, evidence contract, evaluation protocol, risk report, and trace log.

ClaimHarness

Claim table, evidence map, audit report, revision suggestions, and replayable agent trace.

Guided Interview Engine

Local rule-based question routing asks one question at a time, tracks understanding, and surfaces missing information before generating artifacts.

Local-first mode

Default mock mode is deterministic and does not require an API key or external model call.

Run locally

Double-click RUN_PROBLEMBRIDGE_WINDOWS.bat, or run the PowerShell launcher from a cloned repository. The app opens locally in your browser.
git clone https://github.com/RubyYii/ClaimHarness.git
cd ClaimHarness
.\scripts\run_problembridge_ui_powershell.ps1
