中文界面

建模前的问题对齐，输出后的证据审计。

这是一个面向跨学科 AI 项目的本地原型。ProblemBridge 帮助团队在建模前把领域工作流讲清楚；ClaimHarness 在文本或系统输出之后检查 claims 是否有证据支持。

给非 AI 背景用户

先描述工作，不要先写 prompt

从日常任务、判断材料、反复卡住的地方、必须由人确认的决定开始。

生成 AI 可读的任务包

本地向导会把这些描述转成工作流图、任务规格、证据契约和评价协议。

工作流

1
领域工作流

记录真实工作、材料、判断和痛点。

2
ProblemBridge

输出 AI 任务规格、证据契约、评价协议和风险报告。

3
AI / 团队工作

在更清楚的任务边界内建模、写作或设计系统。

4
ClaimHarness

审计主张、证据链接、过度表述和人工复核需求。

功能

ProblemBridge

工作流图、痛点矩阵、概念对齐表、任务规格、证据契约、评价协议、风险报告和 trace。

ClaimHarness

主张表、证据图、审计报告、修改建议和可复现的过程记录。

本地优先

默认 mock mode 是确定性的，不需要 API key，也不会调用外部模型。

本地运行

双击 RUN_PROBLEMBRIDGE_WINDOWS.bat，或在 clone 后运行 PowerShell 启动脚本。界面会在本机浏览器打开。
git clone https://github.com/RubyYii/ClaimHarness.git
cd ClaimHarness
.\scripts\run_problembridge_ui_powershell.ps1

安全边界

不要输入真实患者数据、机密论文、API key、未公开项目材料或敏感个人信息。远程模型接口是可选的，只用于 advisory review。
