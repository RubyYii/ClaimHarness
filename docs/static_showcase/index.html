中文导览

先把领域问题讲清楚，再检查输出有没有证据。

ProblemBridge 面向建模前的问题对齐；ClaimHarness 面向输出后的证据审计。两者合在一起，帮助跨学科团队减少“问题没说清、任务跑偏、结论越界”的风险。

本地优先 默认不需要 API 适合非 AI 背景用户 只使用合成样例

给领域用户

不要一开始就描述“AI 要做什么”。先描述日常工作、判断材料、反复卡住的地方，以及哪些决定必须由人确认。

English guide

Align the domain problem first. Audit the evidence after.

ProblemBridge supports pre-model problem alignment. ClaimHarness supports post-output evidence auditing. Together, they help interdisciplinary teams reduce vague problem framing, task drift, and unsupported claims.

Local-first No API required by default Built for non-AI users Synthetic examples only

For non-AI users

Do not start by describing an “AI task”. Start with repeated work, judgement materials, friction points, and decisions that require human review.
