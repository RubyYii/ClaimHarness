Interdisciplinary AI workflow harness

Align the problem first. Audit the evidence after.

ProblemBridge helps non-AI users start from daily work, judgement materials, pain points, and review boundaries. ClaimHarness then audits claim-evidence alignment, overclaims, and human-review needs after outputs exist.

Local-first No API required by default Built for non-AI users Synthetic examples

For non-AI users

Do not describe an “AI task”. Describe what you repeatedly do, what materials you inspect, where decisions get stuck, and where a human must stay in the loop.

Two-stage workflow

1
Domain workflow

Start from real work, materials, judgement, and pain points.

2
ProblemBridge

Generate a workflow map, AI task spec, evidence contract, and evaluation protocol.

3
AI / team work

Build, write, or design within clearer task boundaries.

4
ClaimHarness

Audit whether claims are supported, overclaimed, or need human review.

Who should test it

Domain practitioners

Useful for clinicians, educators, cultural researchers, and policy researchers who need to explain a workflow before writing an AI prompt.

AI / research users

Useful for people translating domain problems into task specs, evaluation protocols, and evidence boundaries.
